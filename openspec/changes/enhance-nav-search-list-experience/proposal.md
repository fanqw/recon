## Why

当前后台在导航结构、列表检索、主题体验、危险操作入口与测试数据基线上仍不一致，影响日常操作效率与回归测试稳定性。需要在同一变更中完成信息架构对齐、列表能力补齐与数据基线标准化，避免分批改造造成反复回归。

## What Changes

- 侧边栏菜单结构对齐 v1：
  - 工作台（页面占位）
  - 物料管理：商品分类、商品单位、商品信息、进货地
  - 订单管理：订单列表
- 面包屑映射按新菜单层级同步调整。
- 增加搜索能力：
  - 分类、单位：名称、备注（原描述）模糊搜索
  - 商品：名称、分类、单位、备注模糊搜索
  - 进货地：名称、市场名称、备注模糊搜索
  - 订单：名称、进货地、市场名称、备注模糊搜索
- 除订单详情外，所有列表增加创建时间、更新时间字段，并默认按更新时间倒序。
- 列表中的“描述”文案统一改为“备注”。
- 所有列表操作列改为紧凑列宽，并固定在右侧。
- 订单详情页移除“删除订单”，迁移到订单列表操作列（高危红色 + 二次确认）。
- 修复系统内下拉搜索与“使用当前输入”选项能力，恢复可用。
- 参考 Arco Pro，在 header 增加主题切换，且主题仅为 `light / dark`。
- 增加侧边栏收起/展开能力。
- 重置数据库业务数据并统一采用语义化中文样本用于联调与测试：
  - 分类：水果、蔬菜、副食、肉类
  - 单位：斤、件、箱
  - 商品：苹果、生菜、米、面、猪肉
  - 进货地：武汉-中心港市场、洛阳-宏进市场
- 订单详情页详情区域重构为紧凑布局，减少顶部占用，释放明细展示区域。

## Capabilities

### New Capabilities
- `test-fixtures`: 提供可重复执行的中文语义化种子数据基线，支持清库后快速回填。

### Modified Capabilities
- `workspace-navigation`: 菜单结构与面包屑对齐 v1，新增侧栏收起/展开与 light/dark 主题切换。
- `master-data`: 主数据列表搜索、排序、时间字段、备注文案、操作列固定与下拉搜索能力修复。
- `sales-orders`: 订单列表搜索、排序、时间字段、备注文案、操作列固定，删除入口迁移，详情区紧凑化。
- `automated-acceptance`: 验收覆盖导航、搜索、主题切换、删除入口迁移、详情区紧凑化与中文数据基线。

## Impact

- 前端壳层与导航：`apps/web/src/components/dashboard-nav.tsx`、`dashboard-breadcrumb.tsx`、`dashboard-header.tsx`、dashboard layout。
- 列表与详情页面：分类、单位、商品、进货地、订单列表、订单详情。
- 后端查询与交互：相关列表 API 的搜索参数、默认排序与返回字段。
- 通用组件：下拉搜索组件与“使用当前输入”逻辑。
- 数据基线：`apps/web/prisma/seed.ts`。
- 测试与文档：E2E、API 集成测试与 README。
