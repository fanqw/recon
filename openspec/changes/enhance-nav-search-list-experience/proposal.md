## Why

当前后台信息架构与 v1 菜单分组不一致，列表检索能力不足且交互行为存在缺失（如下拉搜索/“使用当前输入”失效），导致高频操作路径变长、可发现性下降。需要一次统一的导航与列表体验修正，使信息组织、检索、操作入口与业务使用习惯对齐。

## What Changes

- 侧边栏菜单调整为分组结构，并与 v1 信息架构对齐：工作台、物料管理（商品分类/商品单位/商品信息/进货地）、订单管理（订单列表）。
- 面包屑导航重构，匹配新的菜单分组层级与路由层级。
- 为分类、单位、商品、进货地、订单列表新增关键字搜索能力：
  - 分类/单位：名称、描述（备注）模糊搜索。
  - 商品：名称、分类、单位、描述（备注）模糊搜索。
  - 进货地：名称（进货地）、市场名称、描述（备注）模糊搜索。
  - 订单：名称、进货地、市场名称、描述（备注）模糊搜索。
- 除订单详情页外，所有列表增加“创建时间”“更新时间”字段，并按“更新时间倒序”作为默认排序。
- 列表中的“描述”字段文案统一替换为“备注”。
- 列表“操作”列统一紧凑化并固定在右侧。
- 订单详情页“删除订单”入口移除，迁移至订单列表页。
- 修复并完善系统内下拉选项搜索及“使用当前输入”类选项能力。
- 参考 Arco Pro，在 header 增加一键主题切换功能。

## Capabilities

### New Capabilities
- `workspace-navigation`: 后台工作区导航结构、面包屑层级映射与主题切换能力。

### Modified Capabilities
- `master-data`: 主数据列表检索能力、列表字段与排序、文案统一、操作列行为、下拉搜索与“使用当前输入”能力修复。
- `sales-orders`: 订单列表检索能力、列表字段与排序、删除订单入口迁移、操作列行为、下拉搜索能力修复。
- `automated-acceptance`: 自动化验收覆盖新菜单结构、搜索能力、主题切换入口及删除入口迁移。

## Impact

- 前端信息架构：`apps/web/src/components/dashboard-nav.tsx`、面包屑组件与 dashboard layout。
- 列表页面与详情页：分类、单位、商品、进货地、订单列表、订单详情相关页面与表格组件。
- API 查询参数与后端查询逻辑：主数据与订单列表接口需支持多字段模糊搜索与统一排序。
- 前端可复用组件：列表表格封装、下拉检索组件、header 主题切换控件。
- 自动化测试：API 集成测试与 E2E 断言需同步更新（新增搜索与入口变更验证）。
