# automated-acceptance

## ADDED Requirements

### Requirement: 核心 API 的自动化验证

系统 SHALL 提供可在本地与 CI 中重复执行的自动化测试，验证主要 HTTP API（或等价服务端入口）在已认证与未认证场景下的行为符合 `admin-authentication`、`master-data`、`sales-orders` 中的关键路径。

#### Scenario: 未认证访问受保护 API 失败

- **WHEN** 测试在未携带会话的情况下调用受保护的业务 API
- **THEN** 测试 SHALL 断言响应为未授权（如 401/403），且套件通过

#### Scenario: 认证后主数据与订单 API 可通

- **WHEN** 测试在有效会话下依次调用分类、单位、商品、订单及订单相关 API 的最小成功路径（创建或查询，与实现阶段约定）
- **THEN** 测试 SHALL 断言 HTTP 成功状态与响应体中存在预期标识或字段，且套件通过

### Requirement: 关键页面的浏览器自动化

系统 SHALL 提供浏览器端端到端测试，验证登录页与主要业务页面上的可交互控件（按钮、链接、表单提交）可被操作，且主流程无阻塞性错误（与实现阶段选定的框架一致，推荐 Playwright）。

#### Scenario: 登录页可交互

- **WHEN** E2E 启动应用并打开登录页
- **THEN** 测试 SHALL 能够填充用户名与密码字段并触发提交控件，且不因前端错误导致流程中断

#### Scenario: 登录后主导航可达且基础页可点击

- **WHEN** E2E 使用有效凭据完成登录
- **THEN** 测试 SHALL 能够导航至分类、单位、商品、订单列表及至少一个订单详情相关路径，并对每页至少一个主要操作控件执行点击（或等价交互）且页面无未捕获致命错误

#### Scenario: 订单详情页导出 Excel 可触发

- **WHEN** E2E 在已登录状态下进入某订单详情页
- **THEN** 测试 SHALL 能够找到并点击「导出 Excel」（或与实现一致的等价按钮文案）且不因前端错误中断；可选进一步断言下载事件或文件扩展名为 `.xlsx`（与实现约定）

### Requirement: 测试运行的文档与入口命令

仓库 SHALL 在 `README.md`（或项目约定位置）中说明如何安装依赖、准备数据库、启动应用，以及执行 API 测试与 E2E 的命令；根目录或 `apps/web` SHALL 提供 npm/pnpm 脚本入口以便一键运行。

#### Scenario: 新贡献者可按文档运行测试

- **WHEN** 贡献者遵循文档中的环境准备步骤并执行文档中写明的测试命令
- **THEN** 其 SHALL 能够在无额外未文档化步骤的情况下启动测试命令（允许配置 `.env` 等已说明项）
