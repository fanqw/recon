# admin-authentication

## ADDED Requirements

### Requirement: 管理员使用用户名与密码登录

系统 SHALL 允许持有有效凭据的管理员通过用户名与密码完成认证，并建立已登录会话，使后续受保护的 API 与页面访问被允许。

#### Scenario: 凭据正确时登录成功

- **WHEN** 客户端使用数据库中存在的用户名与正确密码提交登录请求
- **THEN** 系统 SHALL 建立会话（或等价的已认证状态），且客户端能够访问至少一个受保护的业务 API 或页面而不被重定向到登录页

#### Scenario: 凭据错误时登录失败

- **WHEN** 客户端使用错误密码或不存在的用户名提交登录请求
- **THEN** 系统 SHALL 拒绝建立会话，且不得泄露与「用户是否存在」相矛盾的敏感信息（错误提示可为通用文案）

### Requirement: 未认证访问受保护资源被拒绝

系统 SHALL 确保未登录用户无法执行业务写操作或访问需登录的 API；对页面访问应重定向到登录页或返回等价的未授权响应。

#### Scenario: 未登录调用受保护 API

- **WHEN** 未携带有效会话的请求访问标记为需认证的 API
- **THEN** 系统 SHALL 返回 HTTP 401 或 403（与实现约定一致且文档化），且不得返回成功业务数据

#### Scenario: 未登录访问受保护页面

- **WHEN** 浏览器未登录用户访问需登录的页面路径
- **THEN** 系统 SHALL 将其导向登录页或展示无法继续业务操作的未登录状态

### Requirement: 管理员可登出

系统 SHALL 提供登出能力，使当前会话失效，后续请求视为未认证。

#### Scenario: 登出后会话失效

- **WHEN** 已登录管理员执行登出
- **THEN** 系统 SHALL 使该会话不再有效，且随后对受保护 API 的请求 SHALL 被拒绝
