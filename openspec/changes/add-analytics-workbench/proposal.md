## 背景与动机

当前工作台仍是占位入口，系统已有订单、订单明细、商品分类、进货地等结构化数据，但缺少面向经营复盘的汇总视图。需要提供一个数据工作台，让管理员能按业务维度和时间粒度快速查看金额分布与趋势。

## 变更内容

- 新增数据工作台页面，基于未删除订单与未删除订单明细展示核心统计。
- 使用 ECharts 提供柱形图、折线图、饼图等可视化，用于展示分类、商品、进货地、总金额与时间趋势。
- 支持按天、周、月、年统计金额趋势，并支持按分类、商品、进货地等维度查看排行或占比。
- 明确统计口径：金额默认基于订单明细持久化 `line_total` 汇总，逻辑删除的数据不参与统计。
- 将现有“工作台”导航从占位入口升级为可用的数据概览入口。

## 能力范围

### 新增能力

- `analytics-workbench`: 覆盖数据工作台的统计口径、维度聚合、图表展示、筛选与空状态要求。

### 修改能力

- `workspace-navigation`: 工作区导航中的“工作台”入口需要指向并标识实际数据工作台，而不是占位页。

## 影响范围

- 受影响规范：`analytics-workbench`、`workspace-navigation`
- 受影响应用区域：`apps/web/src/app/(dashboard)/workspace/page.tsx`、工作区导航与面包屑工具、工作台组件，以及可能新增的数据统计 API 路由或服务端数据加载器。
- 受影响数据源：Prisma 模型 `Order`、`OrderCommodity`、`Commodity`、`Category`、`Unit`、`PurchasePlace`。
- 新增依赖：ECharts 及其 React 集成方式，需兼容 Next.js App Router 客户端组件与现有构建约束。
